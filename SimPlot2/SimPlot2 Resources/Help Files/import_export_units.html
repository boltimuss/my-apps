Importing and Exporting Units

Rather than constantly building new units from scratch, units may be pre-built, saved, and then loaded into an existing scenario.

Importing Units

Units that have been previously exported (saved) may be imported as new units. These units are the equivalent of units created using the Units menu but with some pre-filled information.

Use the Units/Import Unit menu item to import a unit.

The basic unit category is assigned upon import (air, surface, etc.). The specific Unit Type and Class is also assigned if it was saved in the export.

Imported units will have their Side set to "Neutral" unless the unit Side was set during export. However, if another unit is selected first, then the imported unit is assumed to be the same side as the selected unit (as if a missile or aircraft was launched).

The location of the imported unit will be set to 0, 0 by default. Just as with creating new units using the Units menu, if the map cursor is displayed, then the imported unit will be located there instead. If another unit is selected first, then the imported unit will be co-located with that unit.

For missiles and torpedoes, the Speed and Range values will also be imported if they were saved. This is a special consideration since missiles usually have a set speed and range, but some torpedoes may have a dual setting.

If Sensors and Weapons circles/arcs were saved, then they too will be imported into the new unit. All Sensors and Weapons are imported as not visible and not filled, but the ranges, arcs and colors are preserved.

Exporting Units

Units created inside a scenario may be saved (exported) for future use. Only some of the unit information is exported.

With a unit selected, use the Units/Export Unit menu item to export the unit.

The following unit information is saved into the export file:

  • the basic unit category (Air, Surface, etc.);
  • the unit's Side;
  • the specific Unit Type;
  • the Class if it's specified;
  • the Name if it's specified;
  • the Sensors and Weapons if there are any.

Missiles and torpedoes will have their Speed and Range exported since they are usually constant. When exporting a missile or torpedo, please take care to specify the full range for that type.

Additional information such as X, Y, Course, Altitude, Depth, Perception, and ES/Sonar bearings are not saved. The exported unit is meant to be generic so that additional units of the same type can be imported as unique scenario elements.
